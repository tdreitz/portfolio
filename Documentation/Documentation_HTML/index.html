A) HTML Structure - top

This theme is a fluid content. It is built on Bootstrap. The Html structure is based on the following page logic:

            < div class="bb-ui-view primary left" >
                < div class="bb-ui-view-container" >
                    < div class="bb-ui-view-content" >
                        Content goes here
                    < /div >
                < /div >
            < /div >

ALl pages should have this HTML structure, so the javascript navigation can work properly. Mind the .primary.left classes to the .bb-ui-view container. You need to tell our js plugin which is your primary view, so it can set up the rest

E) Including the sliders - top

The sliders used in Reset are based on the great "swiperjs" javascript library by "idangero.us".

Adding the slider:

<div class="bb-slider" data-centered-slides="false" data-slides-per-view="1" data-direction-nav=".direction-nav-buttons"> <div class="swiper-container"> <div class="swiper-wrapper"> <div class="swiper-slide"> <div class="image-wrap"> <img src="images/blog-image3.png"> </div> </div> <div class="swiper-slide"> <div class="image-wrap"> <img src="images/blog-image.png"> </div> </div> </div> <div class="direction-nav-buttons"> <a href="#" class="arrow-left prev"></a> <a href="#" class="arrow-right next"></a> </div> </div> </div>

This will add a default slider with 2 slides. As you can see, the opening slider tag consists of a couple of attributes. The main attribute - the class attribute needs these classe "bb-slider". The .bb-slider class is used by the javascript to initialize the js widget.

The data attributes are important here to tell the swiper js how to initialize. You hav various options, if you need control nav or direction nav. In this example, the data-direction-nav attribute specifies the jQuery selector for the nav arrows. There are other options for the sliders.

F) Adding sound - top

You can add sound components to your template in an easy and straight forward way. Here is the exemplary HTML used:

<div class="bb-audio-player" data-src="http://antonibotev.com/themes/banda/wp-content/uploads/2013/10/Clouds.mp3" data-play-pause-button-mode="toggle"> <a href="#" class="play-stop button-play"> <i class="fa fa-play"></i> </a> <div class="bb-track-wrap track-progress-bar"> <a href="#" class="track-progress"> <span style="width:0%"></span> </a> </div> <div class="bb-volume-wrap"> <span class="bb-volume"> <div class="bb-track-wrap volume-bar"> <span style="width:67%;"></span> </div> <a href="#" class="button mute button-mute"><i class="fa fa-volume-off"></i></a> <a href="#" class="button max button-unmute"><i class="fa fa-volume-up"></i></a> </span> </div> </div>

You can add sound to any component, just add the class anme "bb-audio-player" and the data-src attribute, where you have to specify the source of the music file you'd like to play. You will need at least a play button in the container. The play button needs to have the class "button play", so the app will track events on this button.

Now take a look at the other attribute - data-play-pause-button-mode="toggle". If set to toggle, the app will create a pause button (although for this to work, you need to use the exact same html structure for the play button as in the example above) and toggle between the play/pause button, depending on the state of the player.
